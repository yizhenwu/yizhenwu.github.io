一个http的请求过程

即在浏览器输入http://yizhenwu.github.io/2015/09/04/pointeronC/index.html

明确一点，http是应用层协议，是通信双方进行通讯的事先约定的规范。

  1. 建立连接

    • 当我们在浏览器输入一个url请求时，首先需要建立一个socket连接。
    • 又，建立socket连接需要通过IP地址和端口号建立。
    • 所以，需要根据url进行DNS解析出IP地址的过程。【DNS解析过程：在本地机器在配置网络时都会填写DNS，因此本机将会把url发送给这个配置的DNS服务器；如果能够找到响应的url即返回其IP地址，否则将该DNS继续解析该请求发给上级DNS（DNS是一个树状结构），该请求将被一直发送到根并直到获得结果；解析完成后，就获得目标IP地址和端口号，可以进行下一步，建立socket连接】
    • 此外，如果url中不包含端口号，则需要用协议的默认端口号(PORT:8O)。
    • socket(),bind(),connect(),send()。

  2. 请求

    • 连接建立成功后，开始向wed服务器发送请求，该请求一般是GET或POST命令（POST命令用于FORM参数的传递）。
    • GET命令格式： GET 路径/文件名 HTTP/1.0，指出所访问的文件；指出Web浏览器使用的HTTP版本。
      因此，GET命令如下： GET /2015/09/04/pointeronC/index.html HTTP/1.0

  3. 应答

    • Web服务收到请求后，进行处理。从其文档空间搜索子目录/pointeronC的index.html。如果找到该文件，web服务器将该文件传送给相应的Web浏览器。
    • 为了告知浏览器，Web服务器首先传送一些HTTP头信息，然后传送具体的内容（HTTP体信息）。HTTP体信息和HTTP头信息之间以一个空行分开。 HTTP头信息包括：1.HTTP 1.0 200 OK，是Web服务器应答的第一行，列出服务器正在运行的HTTP版本号和应答代码，”200 OK”表示请求完成。2.MIME_Version:1.0，表示MIME类型的版本。3.content_type，指示HTTP体信息的MIME类型，如content_type:text/html表达传送的数据是HTML文档。4.content_length，指示HTTP体信息的长度(字节)。

  4. 关闭连接
    应答结束后，Web浏览器与Web服务器必须断开，以保证其他Web浏览器与Web服务器建立连接。

HTTP组成：HTTP请求(POST和GET)、HTTP响应。

《C和指针》总结

第二章

  1. ANSIC的任何一种实现种均包含两种环境（不必位于同一台机器）：
  • 翻译环境:源代码->可执行的机器指令。
    1. 一个或多个源文件编译转化成目标代码(.o或.obj文件)。包括预处理器处理（预处理器在源代码上执行一些文本操作，如#define的替换以及读入#include包含的文件内容）。源代码解析(判断语句的意思，是产生绝大多数警告和错误的阶段，解析后产生目标代码)。最后优化器(对目标代码进一步优化)
      1. 多个目标文件由链接器链接在一起，形成一个可执行程序。
  • 运行环境:实际执行代码。

第三章

  1. 有符号整型变量范围：INT_MIN,INT_MAX。
  2. 将字符转化为二进制：value=value-'0';
  3. 枚举类型就是指其值为符号常量而不是字面值类型，enum Type {apple, banana, Carton };声明类型为Type，这些变量实际是以整型的方式存储，符号名的实际值都是整数值。apple是0，banana是1，默认时后一个符号比前一个符号大1，意味着可以指定符号的整型值。如enum Type {apple=12, banana=15, Carton=20 };。
  4. 每个变量占用一个特定的位置，变量的值都是存储在计算机内存中，每个内存位置都是由地址唯一确定并引用。指针变量是地址的另一个名字，指针变量的值是另一个变量的地址。Note：指针常量意义不大！
  5. 字符串常量：字符串常量存放在静态数据区，双引号括起来的部分，对字符串常量修改的结果是未定义的，在使用的时候字符串常量实际上是一个指针。如，"Hello", "\aWorld\a", ""，最后一个空的也是字符串常量，也是具有一个NIL终止符字节。
  6. 字符串常量不可以给一个数组赋值。这句话的强调的是赋值。当然char str[]="hello world";这句是可以的，因为这是在数组创建的时候为用静态存储区的"hello world"来初始化数组，即str数组拥有一份hello world拷贝，与字符串常量"hello world"是两个东西。–
    • Why：字符串常量为何也存储在静态存储区呢？
    • Reason：细说静态存储区，静态存储区存储的是静态变量和全局变量。而字符串常量在程序中，只有一份拷贝，但是却可以被多次使用，这和静态变量类似。
  7. typedef关键字，允许为各种数据类型定义新名字。需要注意#define与typedef关键字创建新类型名字的区别，如#define ptr_to_char char*; prt_to_char a,b;，此处只有a被正确声明为一个指针，而b仅仅是字符。
  8. 作用域的概念，即变量只在作用域才可以被使用。对于编译器来说可以分四种作用域：文件作用域、函数作用域、代码块作用域和函数原型作用域。
  9. 链接属性，对于链接器来说可分为三种属性：外部external，内部internal，无none。
    • 没有链接属性的标识符none，表示总是被当做单独的个体，即当该标识符的多个声明，也被当做是多个不同且独立的个体。
    • internal链接属性表示标识符在同一个源文件中的所有声明指的是同一个个体，位于不同文件的其他多个声明军属于不同的实体。
    • external链接属性，即不论声明多少次，在多少个文件，都是同一个实体。
    • Note：修改链接器属性的两个关键字：external和static，分别对应于external链接属性和internal链接属性。
    • static关键字修改属性是有限制的，其只能对缺省为external的属性的声明才可以改变其链接属性为internal链接属性。在其他位置，static关键字是声明为静态变量。
    • 此外，external关键字 声明链接属性需要在标识符第一次被声明时指定。

第四章

  1. break和continue语句，break是永久终止循环，continue是终止本次循环，接下来检测表达式的值，以决定是否继续执行循环。
  2. 条件测试时，当前一个条件不满足时，后一个条件语句就不用执行。

第五章

  1. 移位操作符：<<,左移位；>>,右移位。注意有符号数右移位时用1作为填充。
  2. 位操作符：&，|，^：与，或，异或。
    举例：value=value | 1<<bit_number;将某位置1。value=value & ~(1<<bit_number);将某位置0。

第六-九章

int arr[10];
数组名并不用作指针常量的场合：

  1. 当数组名作为sizeof操作符的操作数时，如，sizeof返回整个数组的长度，而不是指向数组的指针的长度。
  2. 当数组名作为单目操作符&的操作数时，如，取一个数组名的地址所产生的一个指向数组的指针，是一个指向某个指针常量的指针。

下标引用和指针间接访问表达式是一样的

  1. arr[num];等同于*(arr+(num));。

数组和指针一样吗？

  1. 声明数组时，编译器根据声明所指定元素的数量为数组保留内存空间，然后再创建数组名，它的值是一个常量，指向这段空间的起始位置。
  2. 声明一个指针时，编译器只为指针本身保留内存空间，并不为任何整型值预留空间。而且直接使用*b是非法的，因为没有初始化b的指向。

数组名做函数参数时

  1. 由于数组名的值是指向数组第一个元素的指针，因此传递给函数的是一份该指针的拷贝。

字符数组的初始化

  1. 以上两种初始化均可，第一种显得笨拙，对与非常短的字符串，可以这么干。
  2. 第二种是语音标准提供的快速方法，但注意区分"hello"并不是一个字符串常量，它只是第一种的初始化列表的另一种写法。
  3. 因此，"hello"是否为一个字符串常量，取决于是否为用于初始化一个字符数组，那么它就是一个初始化列表，否则它就是字符串常量。

多维数组

  1. 多维数组的存储：是按行存储，右下标率先变化的原则。
  2. 如int matrix[2][3];，可以认为创建一个一维数组，包含二个元素，其中每个元素又是包含3个元素的数组，因此matrix是指向一个包含三个元素的整型数组的指针，即是一个指针的指针。因此，matrix+1是指向该数组的第二个元素，这个元素也是一个包含三个元素的数组。

分析：int (*p)[10];怎么判断p指向什么呢？

  1. 假设是对表达式求值的过程，下标优先级高于间接访问，但是间接访问有括号。因此，p是一个指针，是指向某种类型的指针。
  2. 接下来看下标操作，所以p是指向某种类型的数组的指针。
  3. 对(*p)[10]表达式求值得到一个整型，因此p是一个指向整型数组的指针。

多维数组matrix充当函数参数时，函数的原型为：

  1. int func1 (int (*P)[3]);正确。
  2. int func2 (int p[][3]);正确。
  3. int func3 (int **p);是错误的，因为针对一个多维数组，无法知道调整的长度。和一维数组不一样，虽然一维数组可以这么干。

指针数组

  1. 指针数组是一个数组，其元素类型是执行某种类型的指针。
  2. 如int *api[10];分析如下，假设是对这个表达式进行求值。由于下标优先级高于间接访问，因此首先执行的是下标引用，因此api是某种类型的数组，并且该数组包含10个元素。通过下标在取得一个数组元素后，随即执行的是间接访问操作。间接访问操作后得到的是一个整型值。所以，api是一个数组，其元素类型是指向整型的指针。

字符串函数的使用

  1. char* strcpy(char *dst, char const *src);因此，strcpy函数的dst必须是一个可以修改的参数，因此，dst必须是一个字符数组或者是一个指向动态分配内存的数组的指针。注意，不能是一个字符串常量。此外，如果src比dst大，将会覆盖存储与数组后面的内存空间的值！
  • char dst[] = "hello world";正确。
  • char *dst = (char*)malloc(sizeof(char)*10);正确。
  • char *dst = "error";错误。
  1. int strcmp(char const* s1,char const* s2);该函数是比较两个字符串涉及对两个字符串对应的字符逐个进行比较，直到发现不匹配为止，最先不匹配的为止。
  • 若s1小于s2，返回-1；
  • 若s1大于s2，返回1；
  • 若s1等于s2，返回0；

字符串查找

  1. 查找一个字符：
  • char *strchr(char const *str, int ch);如,查到该字符(第一次出现)，则返回指向该字符的指针(含该字符)。若找不到，则返回NULL。
  • char *strrchr(char const *str, int ch);如,查到该字符(最后一次出现)，则返回指向该字符的指针(含该字符)。若找不到，则返回NULL。
  • 调用如下：
  1. 查找任何几个字符-查找任何一组字符第一次在字符串中出现的位置。
  • char *strpbrk(char const *str,char const *group);函数返回指向str中第1个匹配group中的任何一个字符的字符位置。若找不到，则返回NULL。
  • 调用如下：
  1. 查找一个子串-在字符串中查找一个子串
  • char *strstr(char const* s1,char const *s2);函数在s1中查找整个s2第一次出现的起始位置，并返回一个指向该位置的指针。如果s1中并没有完整匹配s2，则返回NULL。此外，如果第二个参数s2为空，则直接返回第一个参数s1。
  • 在字符串中查找一个子串，返回最后一次出现的位置

以下是电话面试的问题及我的解答：

项目相关

Q1：把我对我项目当成一无所知的人，尽量让我了解这个项目。

A1.1：好，要介绍这个项目，我介绍的逻辑是问题的由来–>面临的挑战–>我的方案–>对比方案。

A1.2：问题背景，我先假设一下，一个大型游戏服务提供商准备提供一个在线游戏服务，在部署这个游戏服务时，是不知道以后在线用户的分布情况的。因此，初始部署可能存在盲目性。当部署位置远离大规模在线用户，那么这大规模的在线用户势必都将通过长路由来获取服务。这就是问题的由来。

A1.3：因此，我要做的就是用户分布感知的虚拟机部署和迁移。

Interrupt1：如何做到用户分布感知？

A1.4：需要对用户分布进行量化。我的量化方法是，获取全网在线用户，统计每个区域的在线用户数量。

Interrupt2：用户分布可以直接通过服务器端，看到我当前在线用户的IP地址，根据此IP所属位置，进行划分很方便，何必要分区域去统计呢？

A1.5：直接通过服务器端的在线IP，确实可以看出，当前用户不同区域的分布情况。因为我要做的是进一步要将这些用户划分到某个数据中心所在区域。

Interrupt3：在做这个项目遇到的比较大的问题是什么？
