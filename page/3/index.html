Notes:

  1. 对reference的所有操作都是对“reference所代表的对象”操作。

  2. 为什么使用reference？

    a. 为使用reference之后，不需要再创建对象的副本，省去了一定的复制工作。

    b. 可实现直接对所传入的对象进行修改。

    c. 如果不想对原对象进行修改，则用const修饰该引用。

  3. 提领一个指针前，一定要对这个指针做证空处理。

栈的地址生长方式

栈的地址生长方式

int function(const int src[16])
{
    int val1=0;
    int dst[16]={0};
    int val2=0;
    const int *psrc=src;
    //问题1：如下代码行输出什么？
    fprintf(stdout,"size of src=%lu\n",sizeof(src));
    //问题2：如下代码有什么副作用？
    for(int i=0;i<=16;i++)//越dst[16]界
    {
        dst[i]=*psrc+i;
        psrc++;
    }
        return 0;
 }

常说：

  1. 栈和堆的区别中，有一点是，栈的地址从高地址到低地址生长；

因此在栈中：

val1------高地址
     bigger than
dst[]-----中地址
    且: &dst[1] < &dst[2] <... < &dst[15]    
     bigger than
val-------低地址

因此：当dst数组溢出的时候，将会是向高地址溢出，因此val1将会被覆盖。

  4. 如果想将所有节点都搜索一遍后结束,那么就不需要释放book[i]=0了,根据需要变化。
