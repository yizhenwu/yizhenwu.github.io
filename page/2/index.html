取变量还是取地址

c++中定义了->运算符，该运算符是把解引用和成员访问两个操作结合在一起。即it->men与(*it).men的含义相同。

因此注意区分：
对数组：
struct str_t
{
long long len;
char data[32];

};
struct data1_t
{
    long long len;
    char data1[2]
    int *data2[2];
};

struct str_t str;
memset((void*)&str,0,sizeof(struct str_t));
str.len=sizeof(struct str_t)-sizeof(int);

struct data_t *pData=(struct data1_t*)&str;
(char*)pData->data[0];//**这个是数值，不是首地址**！！！

(char*)pData->data[0];//**这个是地址**！！！

const变量通过指针的修改问题

const变量通过指针的修改问题

  1. const变量是不允许直接被修改的！
    const int i = 0; i = 10;
  2. 但是const变量却可以在特定条件下通过指针修改！如:
    void main(){
    const int i = 0;
int *j = (int *)&i;
*j = 1;
printf("%d,%d", i, *j); //i=0,j=1
system("pause");
    }
  3. 对2结果i=0说明：因为i是const变量，因此在编译器在i的预处理的时候就进行了替换，编译器只对const变量的值读取一次，所以输出0。
  4. 对2结果j=1说明：因为i地址空间实际存储的值发生了改变。（之所以能改变是因为地址在堆里面）
  5. 新例子：
    const int i = 0;
    void main(){
    int *j = (int *)&i;
*j = 1;
printf("%d,%d", i, *j); 
system("pause");
    }//编译错误
    //因为i在全局区，因此只具有只读属性。

    总结

  6. const全局变量存储在全局存储空间，其值只有可读属性，不能修改；
  7. const局部变量存储在堆栈中，可通过指针修改其值；
  8. const变量在预处理是处理，编译器只对其值读取一次。

关于fork()函数大总结

关于fork()函数大总结

基本概念

进程：包括代码、数据和分配给进程的资源（进程是在数据集合上一次的运行）。

fork()：通过系统调用的方式，创建一个与原来(父)进程一模一样的进程，也就是两个进程可以完成同样的处理，但是初始参数不同，进程的处理也就会不同。

系统调用fork()函数后所进行的操作：

  1. 系统首先给新的进程分配资源（如存储数据和代码的空间）。
  2. 把原来(父)进程的值复制到新进程中，copy itself，只有少数的值不同。
  3. 两个进程创建后是独立的，在不同的内存地址中，不共用。
  4. 两个进程的进程号pid不同！
  • 原(父)进程，fork()返回新创建的子进程的进程pid。
  • 子进程中，fork()返回0。
  • 如果出现错误，fork()返回负数：
    • 进程数达到上限；
    • 内存不足。

举例说明1

//代码说下
int main(　）{ 
     int i; 
     for(i = 0; i<5;i++){ 
        int pid = fork(); 
        if(pid == 0){ 
            //do something 
        } else { 
        //do something 
        } 
        }
    // do somthing,sleep
    return 0;
}

注意：

  1. 在执行int pid = fork()之前只有一个进程在执行代码。
  2. 但在这句之后将会变成两个进程，这两个进程一模一样，均执行下一句if(pid == 0)...后面的代码。

参考：linux中fork（）函数详解（原创！！实例讲解）

举例说明2

int main()
{
    pid_t pid1;
    pid_t pid2;

    pid1 = fork();
    pid2 = fork();
    printf("pid1:%d, pid2:%d\n", pid1, pid2);
}

要求：这个程序执行到这个程序的所有进程结束这段时间，没有其他新进程执行。

  1. 一共运行了几个进程？
  2. 如果其中一个进程输出“pid：1001，pid2：1002”，那么其他进程输出结果(不考虑执行顺序)？

预备知识

  1. 在Linux下，进程有一个唯一的PID标识进程。PID从1~32768的正整数，其中init进程的pid=1，其他进程都是从2以此编号，用完32768之后，从2重新开始。
  2. Linux进程查询 ps aux 或者 top命令。Linux的进程采用进程表结构存储正在运行的进程。
  3. Linux的进程呈树状结构，init是所有进程的父进程，也是第一个进程。

详细细节

如图所示fork()进程：

  1. 执行程序，生成进程P，P执行完所有的Part.A的代码。
  2. 当执行到pid=fork();时，P启动一个新的进程Q，Q是P的子进程且和P是一个程序的进程。Q由P直接copy itself生成（拥有其所有变量、计数器等当前值）。
  3. P进程中，fork()将Q的PID放回给变量pid，并继续执行Part.B的代码。
  4. Q进程中，将o赋给pid，并执行Part.B代码。

关键细节

  1. P进程执行了所有的程序，而Q进程只执行了fork()后面的所有程序(Part.B)。
  2. Q拥有P进程的运行变量及环境等，在这个基础上运行，而不是初始环境。
  3. P中的fork()启动子进程Q，返回Q的pid。而Q的fork()不启动新的进程，将返回o。

解题

进程树如图：

  1. 假设初始进程是P0，设其PID=xxx；
  2. 当执行到pid1=fork();时，P0将启动新的子进程P1，题目中P1的PID为1001。
  3. 因此，P0的fork()返回1001给pid1，将继续执行pid2=fork();此时启动一个新进程P2，题目中P2的PID=1002。
  4. 因此，P0中第二个fork返回1002给pid2，继续执行完后续程序，直到结束。所以程序输出“pid1:1001,pid2:1002”。
  5. 此时分析P2，P2启动时，P0中pid=1001，所以P2复制P0的1001，而作为子进程pid2=o。从P2的第二个fork()后开始执行，结束后输出“pid1:1001,pid2:o”。
  6. 此时分析P1，P1的第一条fork()返回o给pid1，然后执行后面的代码。下一句代码即pid2=fork();此时P1产生新进程P3。
  7. P1的pid2=fork();返回PID:1003给pid2，所以pid2=1003。P1继续执行后续程序，结束，输出“pid1:o,pid2:1003”。
  8. P3作为P1的子进程，复制P1中pid1=o，并且第二条fork()返回o给pid2，所以P3最后输出“pid1:o,pid2:o”。
  9. 程序结束。

switch语句与break分析

例子

//程序中没有break，分析程序输出。
for(int i=0;i<3;i++)
{
    switch(i)
    {
        case 0:printf("%d",i);
        case 2:printf("%d",i);
        default:printf("%d",i);
    }
}

分析

  1. i=0:
    • case 0:printf(“%d”,i); 0
    • case 2:printf(“%d”,i); 0
    • default:printf(“%d”,i); 0
  2. i=1:
    • default:printf(“%d”,i); 1
  3. i=2:
    • case 2:printf(“%d”,i); 2
    • default:printf(“%d”,i); 2

结论

对于满足条件已经进入分支，case语句没有break，分支后面的语句都是会被执行，包括default。

C++内存管理

32位/64位系统中不同类型变量对应的字节数

  1. char: 1字节；
  2. char*: 4字节(32位)/8字节(64位)；
  3. short int: 2字节；
  4. int: 4字节；
  5. unsigned int: 4字节；
  6. float: 4字节；
  7. double: 8字节；
  8. long: 4字节(32位)/8字节(64位)；
  9. long long: 8字节；
  10. usingned long: 4字节(32位)/8字节(64位)；

内存管理

64位系统中内存管理遵循8字节对齐

原则：按照8字节为单位分配存储空间，如果不足，则自动补齐，本次分配不足以存放下一个变量时，会重新分配空间。

struct A{
   unsigned int a;//a占4字节,开辟一个8字节内存P;
   char b[2];//b[]占1*2=2字节,在a后的连续2字节内存放，
             //P还剩2字节;
   double c;//c占8字节,P不足以存放,因此将剩余的2字节补全
            //因此开辟新8个字节存放c
   short d;//d占2字节，开辟新的8字节内存，存放d。
           //将剩余6字节补全.
}

So：总共开辟了3个8字节的空间，即24字节空间来存储该结构。

栈的地址生长方式

栈的地址生长方式

int function(const int src[16])
{
    int val1=0;
    int dst[16]={0};
    int val2=0;
    const int *psrc=src;
    //问题1：如下代码行输出什么？
    fprintf(stdout,"size of src=%lu\n",sizeof(src));
    //问题2：如下代码有什么副作用？
    for(int i=0;i<=16;i++)//越dst[16]界
    {
        dst[i]=*psrc+i;
        psrc++;
    }
        return 0;
 }

常说：

  1. 栈和堆的区别中，有一点是，栈的地址从高地址到低地址生长；

因此在栈中：

val1------高地址
     bigger than
dst[]-----中地址
    且: &dst[1] < &dst[2] <... < &dst[15]    
     bigger than
val-------低地址

因此：当dst数组溢出的时候，将会是向高地址溢出，因此val1将会被覆盖。
