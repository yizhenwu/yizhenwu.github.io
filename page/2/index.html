关于fork()函数大总结

关于fork()函数大总结

基本概念

进程：包括代码、数据和分配给进程的资源（进程是在数据集合上一次的运行）。

fork()：通过系统调用的方式，创建一个与原来(父)进程一模一样的进程，也就是两个进程可以完成同样的处理，但是初始参数不同，进程的处理也就会不同。

系统调用fork()函数后所进行的操作：

  1. 系统首先给新的进程分配资源（如存储数据和代码的空间）。
  2. 把原来(父)进程的值复制到新进程中，copy itself，只有少数的值不同。
  3. 两个进程创建后是独立的，在不同的内存地址中，不共用。
  4. 两个进程的进程号pid不同！
  • 原(父)进程，fork()返回新创建的子进程的进程pid。
  • 子进程中，fork()返回0。
  • 如果出现错误，fork()返回负数：
    • 进程数达到上限；
    • 内存不足。

举例说明1

//代码说下
int main(　）{ 
     int i; 
     for(i = 0; i<5;i++){ 
        int pid = fork(); 
        if(pid == 0){ 
            //do something 
        } else { 
        //do something 
        } 
        }
    // do somthing,sleep
    return 0;
}

注意：

  1. 在执行int pid = fork()之前只有一个进程在执行代码。
  2. 但在这句之后将会变成两个进程，这两个进程一模一样，均执行下一句if(pid == 0)...后面的代码。

参考：linux中fork（）函数详解（原创！！实例讲解）

举例说明2

int main()
{
    pid_t pid1;
    pid_t pid2;

    pid1 = fork();
    pid2 = fork();
    printf("pid1:%d, pid2:%d\n", pid1, pid2);
}

要求：这个程序执行到这个程序的所有进程结束这段时间，没有其他新进程执行。

  1. 一共运行了几个进程？
  2. 如果其中一个进程输出“pid：1001，pid2：1002”，那么其他进程输出结果(不考虑执行顺序)？

预备知识

  1. 在Linux下，进程有一个唯一的PID标识进程。PID从1~32768的正整数，其中init进程的pid=1，其他进程都是从2以此编号，用完32768之后，从2重新开始。
  2. Linux进程查询 ps aux 或者 top命令。Linux的进程采用进程表结构存储正在运行的进程。
  3. Linux的进程呈树状结构，init是所有进程的父进程，也是第一个进程。

详细细节

如图所示fork()进程：

  1. 执行程序，生成进程P，P执行完所有的Part.A的代码。
  2. 当执行到pid=fork();时，P启动一个新的进程Q，Q是P的子进程且和P是一个程序的进程。Q由P直接copy itself生成（拥有其所有变量、计数器等当前值）。
  3. P进程中，fork()将Q的PID放回给变量pid，并继续执行Part.B的代码。
  4. Q进程中，将o赋给pid，并执行Part.B代码。

关键细节

  1. P进程执行了所有的程序，而Q进程只执行了fork()后面的所有程序(Part.B)。
  2. Q拥有P进程的运行变量及环境等，在这个基础上运行，而不是初始环境。
  3. P中的fork()启动子进程Q，返回Q的pid。而Q的fork()不启动新的进程，将返回o。

解题

进程树如图：

  1. 假设初始进程是P0，设其PID=xxx；
  2. 当执行到pid1=fork();时，P0将启动新的子进程P1，题目中P1的PID为1001。
  3. 因此，P0的fork()返回1001给pid1，将继续执行pid2=fork();此时启动一个新进程P2，题目中P2的PID=1002。
  4. 因此，P0中第二个fork返回1002给pid2，继续执行完后续程序，直到结束。所以程序输出“pid1:1001,pid2:1002”。
  5. 此时分析P2，P2启动时，P0中pid=1001，所以P2复制P0的1001，而作为子进程pid2=o。从P2的第二个fork()后开始执行，结束后输出“pid1:1001,pid2:o”。
  6. 此时分析P1，P1的第一条fork()返回o给pid1，然后执行后面的代码。下一句代码即pid2=fork();此时P1产生新进程P3。
  7. P1的pid2=fork();返回PID:1003给pid2，所以pid2=1003。P1继续执行后续程序，结束，输出“pid1:o,pid2:1003”。
  8. P3作为P1的子进程，复制P1中pid1=o，并且第二条fork()返回o给pid2，所以P3最后输出“pid1:o,pid2:o”。
  9. 程序结束。

TCP拥塞控制

TCP拥塞控制

什么是拥塞

对网络的某一资源的需求超过资源的可用范围，网络性能变坏,这就是拥塞。

区分拥塞控制和流量控制

  • 拥塞控制:全局性的过程,防止过多的发送方数据注入网络中,避免路由器和链路不发生过载;
  • 流量控制:点对点的通信量控制。

拥塞控制对象

拥塞的原因即负载过大,因此拥塞控制的对象就是发送者的流量(源头),TCP的滑动窗口协议,窗口大小取决于接收窗口和拥塞控制窗口。区分:发送窗口 <= 拥塞窗口
发送窗口=min(rwnd,cwnd)

拥塞控制方法

发送方维护一个拥塞窗口cwnd(congestion window)的状态变量。拥塞窗口取决于网络的拥塞情况,并且动态变化。

  • 慢启动
    思路:一开始不发送大量的数据,先探测网路的拥塞程度,因此从小到大增加拥塞窗口。假定每个TCP报文都按照MSS最大报文长度来发送。
    Start ——> cwnd=1;
    after 1 RTT ——> cwnd=2;
    after 2 RTT ——> cwnd=4;
    after 3 RTT ——> cwnd=8;
    …
    慢启动实质是起点低,但是增长速度很快(指数级)。RTT*log2W时间即可占满W带宽(假设允许无线增长下去,当然不允许这样下去)。

  • 慢启动门限(ssthresh)
    即:当cwnd超过该门限,慢启动过程结束,进入拥塞避免阶段。

  • 拥塞避免阶段做法
    化指数增长为加法增长,此时每收到一次确认:cwnd=cwnd+1。慢慢的将吞吐量调整到最大值。

  • 拥塞检测
    TCP认为网络拥塞的主体是拥塞丢包,即每个TCP报文都有一个定时器(RTO:重传定时器),当RTO超时还没有得到ACK,TCP就对该报文进行重传,而且TCP即认为是网络出现拥塞了:
    ++ 把ssthresh降低为cwnd的一半;
    ++ 把cwnd设置为1;
    ++ 重新进入慢启动过程。

问题回忆罗列

First.自我介绍?

Q1:开始问我的项目，问我最想分享的一个项目是?
A1:说了我最近一年都在做的虚拟机迁移的项目,在项目中着重描述了我解决问题的出发点,随后陈述了两个基本问题，最后给出我的方案。

Q2:SDN网路和IP网络最大的不同是?
A2:关于这个的不同:我觉得主要是SDN网络核心思想就是将控制与数据分离,采用集中控制的方式,而IP网络中路由是由路由器分布式路由。

Q3:C++和C有什么不同?
A3:C++相对C来说，更加具有封装性，支持重载、继承和多态等特性。

Q4:在平时写C++程序中,用到了重载和多态吗?觉得重载有什么问题?
A4:重载使用多了，会导致程序看起来比较混乱(不怎么会答)。

Q5:问一个算法:给出两个int数组,两个int数组元素都不存在重复,如何找出两个数组相同的元素,并且给出算法的复杂度分析。
A5.1:首先枚举是可以的，但是复杂度是O(n^2);
A5.2:设计一个位存储,将两个数组的元素读入数组。

Q6:这样做的缺点是?有没有别的方法?
A6.1:当一个数很大?将会占用很长的bit位。
A6.2:可以用查找的方式,我说的做法是,取较长数组A对元素进行hash之后,用另一数组B查找该元素是否在,hash查找的复杂度为O(1),因此查找效率很好。

Q7:hash表实现的几种方式你知道吗?
A7:面试我的时候，我当时不知道,得补!!!

Q7:你平时编程都用到了哪些数据结构?
A7:我用过图的数据结构,链表和数组等。这个他没有接着去问了~

Q8:考虑到我所关注的大多是网络层,现在以我对网络层的理解,怎么看待http协议的优缺点?
A8:首先我觉得http协议是client发起请求->server,server响应,我觉得这是一种单工的操作方式,也就是服务器端不能主动推送消息至client,而网络层的交互可以看成是“双工”。

Q8.1:这个是一个特点,还有别的吗?
A8.1:我不知道了。。。得补!!!

Q9:再问一个C和C++内存管理,我是怎么理解的?
A9:首先C++允许使用C中malloc和free函数,同时C++有增加内存管理的操作符new和delete,是的可以为对象的内存管理。。。

Q10:在多线程场景下,多线程操作一块内存,那怎么样控制new和delete的匹配?
A10:我思考了下,加一个flag标识。当这块内存被new出来之后,就置flag=1,在释放内存的时候,使用这个内存的线程检查该flag再释放内存。

Q11:TCP的拥塞控制?
A11:通过窗口机制实现拥塞控制,具体的我也没答出来。

Q12:我怎么看待TCP和UDP?
A12:我觉得TCP和UDP本质区别是TCP是有连接的,这种连接实质上是通信双方维护一致的连接状态而看起来是有连接的,而UDP是无连接的。由此点可以引出,TCP连接确保数据包有序,但是确保有序付出的代价是吞吐量总是波动的。相对UDP的尽力传输机制就吞吐量来说优势就不明显了。

Q13:移动网络中访问一个网站和wifi网络下访问一个网站有何区别?
A13:这个问题我不会。。。

Q14:IP数据包的包头都有哪些字段?
A14:我说我知道有MTU,TTL,说了MTU,TTL的功能…问的太细致了!!!
